# SGraph <-> Omega Connector Contract (v1)

## 1. Scope

This document describes the runtime contract between:

- SGraph-side trusted flow (caller of connector),
- `sgraph-omega-connector` API,
- Omega API (upstream scanner).

This contract is valid for API `v1` and is aligned with:

- `contracts/openapi/connector-v1.yaml`
- `connector/models.py`
- `connector/auth.py`
- `connector/omega_client.py`

## 2. Runtime Flow

1. SGraph flow obtains decrypted content in trusted context.
2. SGraph (or trusted worker) sends payload to `POST /v1/scan/attachment`.
3. Connector validates `X-API-Key` + HMAC signature + replay protection.
4. Connector forwards request to Omega `POST /v1/scan/attachment`.
5. Connector returns normalized envelope:
   - `request_id`, `tenant_id`, `risk_score`, `verdict`, `reasons`, `evidence_id`, `policy_trace`, optional `attestation`.

## 3. Request Contract (`POST /v1/scan/attachment`)

Headers:

- `X-API-Key` (required)
- `X-Signature` (required when HMAC enabled)
- `X-Timestamp` (required when HMAC enabled)
- `X-Nonce` (required when HMAC enabled)
- `Content-Type: application/json`

Body:

- `tenant_id` (required)
- `request_id` (optional, generated by connector if missing)
- `filename` (optional)
- `mime` (optional)
- exactly one or both content fields:
  - `file_base64`
  - `extracted_text`
- `metadata` (optional map, additive-safe)

## 4. Response Contract

Success (HTTP 200):

- `request_id: string`
- `tenant_id: string`
- `risk_score: int (0..100)`
- `verdict: allow|quarantine|block`
- `reasons: string[]`
- `evidence_id: string`
- `policy_trace: object`
- `attestation: object|null` (optional)

Errors (`400|401|403|409|500`):

- `request_id: string|null`
- `error.code: string`
- `error.message: string`
- `error.details: object`

## 5. HMAC Canonical String

Canonical string (newline-separated):

1. HTTP method (uppercase)
2. request path (for example `/v1/scan/attachment`)
3. SHA256 hex of raw request body bytes
4. `tenant_id`
5. `request_id` (empty string if absent in body)
6. `X-Timestamp`
7. `X-Nonce`

Signature:

- `base64url(hmac_sha256(secret, canonical_string))`

## 6. Replay and Time Window

- Replay key: tenant + api-key-hash + nonce.
- Nonce TTL: `NONCE_TTL_SEC` (default `600`).
- Max clock skew: `MAX_CLOCK_SKEW_SEC` (default `300`).

## 7. Fallback Behavior

On Omega upstream failures connector returns a safe fallback decision (default `quarantine`), with reason:

- `omega_timeout`
- `omega_unavailable`
- `omega_rejected_4xx`
- `invalid_response`

No plaintext persistence is performed by connector; only redacted audit logging is allowed.

## 8. Contract Source of Truth and Validation

Primary source:

- `contracts/openapi/connector-v1.yaml`

Validation:

- `make test-contracts`
- `pytest -q tests/e2e/test_openapi_contract.py`

If OpenAPI and runtime diverge, OpenAPI and tests must be updated in the same change set.
